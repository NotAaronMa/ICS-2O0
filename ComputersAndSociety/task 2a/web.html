With the prevelance of the internet, more and more importatant events such as finance are being done on computers, network security has becoming a far more relevant concern. Nowadays, a skilled hacker can bring down an entire town's social services or leak the private data of millions of people. Therefore, being knowledgable of possible attacks and how to avoid them is an important skill in the 21st century.

The attacks listed above should never be done on a server unless expliced permission is given by the owner, failure to comply is in defiance of the Computer Fraud and Abuse act and can be punished with up to 20 years in prison or a $15000 fine.

The first attack that we will be covering is a denial of service attack.This is done by getting many computers to sent a large amout of ping or POST requests to a server. The server will have to handle all these requests and will most likely go down. One way to get around these attacks are to use a firewall to limit acess to your server using a firewall. A firewall uses techniques such as a packet filter, a proxy server or blocking certain protocals in order to keep a network secure. A DDOS Attack does not require much skill to pull off and can be done by almost anyone who has the required software.

Another common attack is the SQL injection. an SQL injection is a technique used to get a server to execute an SQL query. This vunrability is caused when input to a database isn't sanitized allowing a hacker to inject code into the query making the server excecute unintended commands. For instance, typing in
" DROP TABLE a; ""
will cause a server to delete the table named "a" someone who knows SQL can use this to wreak havoc on a server making it delete all tables or read all the data stored in the database. The way to avoid this is to sanatize inputs. By having a script remove html tags and sql queries fromS the input, one can nullify this vurnability.

An XSS attack is another common attack where the user embedds scripts in his input. For a user may embed a <script> tag in their input. For instance, if a server uses this simple script to display the latest comment a user made:

This is because his input is directly written into the markup of the website. script.js could be used for Phishing, collecting data or auth cookies. Much like the SQL injection the XSS attack can be prevented using input sanitation. by using javascript to check the input data for html tags, this vunrability can also be patched.

In conclusion, The prevention of many common cyber attacks is dilligence and making sure that the end user Does not have too much control of the server. This can be done using methods such as firewalls and input sanitation.
